git-moving-subdir-from-one-git-repository-to-another-preserving-history

Posted on 2018-08-02 Edited on 2020-05-07 In Tools , Git Views:
Symbols count in article: 2.4k Reading time ≈ 2 mins.

调整 git 仓库，将一个仓库中的子文件夹一定到另一个仓库中，目录结构层次不变

Befor

repositoryA/
.........../directoryToKeep
.........../otherDirectory
.........../someFile.ext
repositoryB/
.........../someStuff

After

repositoryA/
.........../otherDirectory
.........../someFile.ext
repositoryB/
.........../someStuff
.........../directoryToKeep

流程如下

首先处理源文件，及文件夹 directoryToKeep，查到的资料使用命令：

git filter-branch --subdirectory-filter directoryToKeep -- --all

进行处理，这样处理会有一个文件，会编程 root 目录，命令解释如下

--subdirectory-filter <directory>
         Only look at the history which touches the given subdirectory. The result will contain that directory (and only
         that) as its project root. Implies the section called “Remap to ancestor”.

因此如果需要保持目录层次不变，需要从其他方面入手，思路是删除其他文件夹的记录，只保留需要的文件夹

repositoryA 操作流程如下

git clone repositoryA
cd repositoryA
git co -b split
git filter-branch --force --index-filter 'git rm --cached --ignore-unmatch otherDirectory someFile.ext -otherDirectory someFile.ext -r-name-filter cat -- --all
git filter-branch --subdirectory-filter . -- --all
git log -p // check log

经过上述处理之后 log 信息很可能包含了 merge branch 信息，可以使用如下命令进行清除

$ git filter-branch -f --prune-empty --parent-filter `pwd` /test.rb split
$ cat test.rb
#!/usr/bin/ruby
old_parents = gets.chomp.gsub('-p ', ' ')

if old_parents.empty? then
  new_parents = []
else
  new_parents = `git show-branch --independent #{old_parents}` .split
end

puts new_parents.map{|p| '-p ' + p}.join(' ')

repositoryB 操作如下

$ git clone repositoryB
$ cd repositoryB
$ git co -b work
$ git remote add split PATH/TO/repositoryA
$ git fetch split
    From ../repositoryA
    * [new branch]        split      -> split/split
    * [new branch]        work       -> split/work
$ git co -b split split/split
    Branch split set up to track remote branch split from split.
    Switched to a new branch 'split'
$ git log --oneline // check log
$ git co work
$ git merge split --allow-unrelated-histories
